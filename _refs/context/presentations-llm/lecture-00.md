## Slide 6 — Laboratory Work — Project Creation

During laboratory work you will design and implement a chosen system.
- For this you will use various CASE tools.

3 project checkpoints + optional project presentation.

You will work in teams.
- Team size: ~4 people.

The project topic must be agreed upon with the laboratory work instructors.

A report must be uploaded to Moodle for each project checkpoint.

---

## Slide 7 — Laboratory Work Objective

To model and implement a working prototype, selecting appropriate tools for different project stages.

During the first laboratory work session you will need to form teams and agree on your team's project topic.

---

## Slide 8 — Project Topic

**Requirements for the system being developed:**
- Must have a database
- Must have a server and a client-side component
- At least 2 different types of system users
- Must use at least one external system

**Examples:**
- Warehouse management system
- Cleaning services platform
- Restaurant management information system
- Car service management system
- Package delivery information system
- Sports tournaments and matches organization website
- Centralized smart home management system
- Computer parts listings system renkupc.lt

---

## Slide 9 — Tools

- For creating diagrams — **Magic Systems of Systems Architect**
- For implementation, any programming language may be used, but **object-oriented programming** must be applied and the architecture must be based on **MVC** principles or an equivalent architecture
- For testing code generation — **Java** language
- For team collaboration on UML models — **Teamwork Cloud**

---

## Slide 10 — 1st, 2nd, and 3rd Project Checkpoints

**Report** — shared for the whole team, must be uploaded to Moodle in advance.

**Defense** — individual for each student, with their Magic project file open; for the 2nd and 3rd checkpoints — also with the implemented system running.

- Defense questions:
  - On the created models, UML diagram elements, and usage of Magic features;
  - For the 2nd and 3rd checkpoints — also on the implemented solution.
- **Each person must have created all types of diagrams!**

## Slide 13 — Project Presentation (Lecturer A. Ušaniov)

Grading criteria (each worth 20%):

- Presentation duration 5 min, coherent speech, slide professionalism — **20%**
- Accurately, substantively, concisely, and clearly present the work objective — **20%**
- Highlight key architectural decisions with examples — **20%**
- Express in numbers the process performance, system scope, data flows — **20%**
- Demonstrate the created prototype in operation — **20%**
