# sync-docs workflow

**Goal:** Full autonomous resync of all project documentation using bmad-document-project (deep scan) followed by a targeted source exploration pass to patch anything still stale.

**When to use:** Docs are significantly behind — multiple refactor commits, structural changes, new modules. Not for small incremental updates (the post-commit hook handles those).

**Model:** Use Opus for the subagent.

---

## Step 1: Read context + ask scan level

Before spawning anything, read:
- `_mind/mind.md` — what has changed recently and why
- `_bmad/bmm/config.yaml` — BMAD config (project_knowledge path, user_name, etc.)

Then ask the user:

```
Choose scan depth:

1. Quick  (2-5 min)   — pattern-based, no source reading. Good for early-stage / speccing.
2. Deep   (10-30 min) — reads critical dirs (src/, scripts/, etc.). Good for catching up after a refactor.
3. Exhaustive (30-120 min) — reads every source file. Full audit.

[1/2/3] (default: 1):
```

Wait for the answer. Store the chosen level as {scan_level} (quick / deep / exhaustive). Default to quick if the user just hits enter.

---

## Step 2: Spawn the bmad-document-project subagent (autonomous)

Use the Agent tool to spawn a subagent with the following prompt. This pre-answers all of bmad-document-project's interactive questions so it runs fully autonomously.

**Subagent prompt to inject:**

```
You are running bmad-document-project for the cvt-scanner project in fully autonomous mode.

Configuration (from _bmad/bmm/config.yaml):
- project_name: cvt-scanner
- user_name: Saulius
- communication_language: English
- document_output_language: English
- project_knowledge: docs/
- planning_artifacts: _bmad-output/planning-artifacts
- output_folder: _bmad-output/

Pre-selected answers to all interactive steps — do NOT ask for user input at any point:
- Existing docs found? → Select option 1: "Re-scan entire project"
- workflow_mode = "full_rescan"
- scan_level = "{scan_level}"  (use the level the user selected in Step 1)
- autonomous = true
- Resume prompt (if state file exists and is recent) → Select option 2: "Start fresh"

Your task:
1. Read and follow: .claude/skills/bmad-document-project/workflows/full-scan-workflow.md
2. Wherever you encounter an <ask> block, use the pre-selected answers above — never pause for user input
3. Perform a deep scan of the entire project, focusing on:
   - src/ (handlers, runtime, storage, common, config, bootstrap.py)
   - scripts/
   - observability/
   - infra/
   - docker-compose*.yml
   - test/ structure
4. Generate or update all documentation in docs/ to accurately reflect the current codebase
5. When complete, print: ✓ bmad-document-project complete — files updated: {list}
```

Wait for the subagent to complete before proceeding to Step 3.

---

## Step 3: Targeted source exploration pass

After the bmad subagent finishes, spawn a second Opus subagent to patch anything it may have missed or gotten wrong.

**Subagent prompt:**

```
You are doing a targeted documentation accuracy pass for the cvt-scanner project.

The docs in docs/ were just regenerated by bmad-document-project. Your job is to
verify and patch the handler-level diagrams and architecture docs against the actual
source code. Be surgical — only change what is wrong or missing.

Step 1 — Read context:
- _mind/mind.md (what changed recently)
- docs/architecture.md
- docs/diagrams/handlers/shallow-scan.md
- docs/diagrams/handlers/deep-scan.md
- docs/diagrams/handlers/indexer.md
- docs/diagrams/handlers/recovery.md
- docs/diagrams/system/runtime.md
- docs/diagrams/storage/database.md

Step 2 — Read the actual source for each domain:
- src/handlers/github/ (all .py files — shallow.py, deep.py, indexer.py, core/, utils/)
- src/handlers/recover/ (handler.py, recovery/)
- src/runtime/ (all .py files)
- src/storage/ (database.py, db/, ops/)
- src/common/ (all .py files)

Step 3 — For each doc file, check:
- Do the described classes/functions match what's actually in the code?
- Are the Mermaid diagrams (if any) accurate to the real call flow?
- Are module paths correct (e.g. src/handlers/github/core/ not src/handlers/github/scanners/)?

Step 4 — Patch only what is wrong. Do not rewrite sections that are accurate.
         Do not add new sections. Do not expand existing ones.

Step 5 — Print: ✓ exploration pass complete — patches applied: {list or "none"}
```

---

## Step 4: Confirm

After both subagents complete, print a summary:

```
✓ sync-docs complete
  — Phase 1 (bmad deep scan): {outcome}
  — Phase 2 (exploration pass): {outcome}
  — Docs updated: docs/
```
